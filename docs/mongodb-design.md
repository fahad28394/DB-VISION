# MongoDB Database Design

## Overview

DBVision uses MongoDB as the main database. MongoDB stores data in collections and documents instead of tables and rows.

The main collections in DBVision are:

- Users
- Servers
- Metrics
- Alerts
- AuditLogs

---

## Users Collection

Purpose:
Stores all users who can access the DBVision application.

Example document:

```json
{
  "_id": "ObjectId",
  "username": "Fahad",
  "email": "fahad@example.com",
  "password": "hashed_password",
  "role": "ADMIN",
  "createdAt": "2026-06-16",
  "updatedAt": "2026-06-16"
}
```

Roles:

- ADMIN: Full system access
- DBA: Monitor databases and manage servers
- VIEWER: Read-only dashboard access

---

## Servers Collection

Purpose:
Stores database servers that will be monitored.

Example document:

```json
{
  "_id": "ObjectId",
  "serverName": "SQLSERVER-PROD-01",
  "databaseType": "SQL Server",
  "host": "192.168.1.20",
  "port": 1433,
  "environment": "PRODUCTION",
  "status": "ONLINE",
  "createdBy": "userId"
}
```

Relationship:

One User can create multiple Servers.

users.\_id
|
|
servers.createdBy

---

## Metrics Collection

Purpose:
Stores performance statistics collected periodically by the monitoring service.

Examples:

- CPU Usage
- Memory Usage
- Disk Usage
- Active Connections
- Database Size

Example document:

```json
{
  "serverId": "serverId",
  "cpuUsage": 45,
  "memoryUsage": 60,
  "diskUsage": 70,
  "connections": 120,
  "databaseSize": 5000,
  "createdAt": "2026-06-16T10:00:00"
}
```

Relationship:

One Server can have millions of metric records.

servers.\_id
|
|
metrics.serverId

---

## Alerts Collection

Purpose:
Stores warnings generated by the monitoring engine.

Example:

- CPU usage above 80%
- Disk usage above 90%
- Backup failure

Example document:

```json
{
  "serverId": "serverId",
  "severity": "CRITICAL",
  "message": "CPU reached 95%",
  "status": "ACTIVE",
  "createdAt": "2026-06-16"
}
```

Alert Status:

- ACTIVE
- RESOLVED
- IGNORED

---

## AuditLogs Collection

Purpose:
Tracks all important user actions.

Examples:

- User login
- Server creation
- Server deletion
- Alert modification

Example document:

```json
{
  "userId": "userId",
  "action": "CREATE_SERVER",
  "description": "Created SQLSERVER-PROD-01",
  "createdAt": "2026-06-16"
}
```

Relationship:

users.\_id
|
|
auditLogs.userId
